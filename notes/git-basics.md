Git - это система контроля версий программ/программного кода 
Репозиторий - это хранилище програмного кода 
Коммиты нужны для отслеживания изменений кода
## Что такое Pull Request?

Pull Request (PR) - это механизм в GitHub/GitLab, который позволяет:
- Предложить изменения в коде
- Обсудить эти изменения с командой
- Провести код-ревью
- Автоматически проверить качество кода
- Безопасно внедрить изменения в основную ветку
## Зачем нужны ветки?

Ветки в Git необходимы для:
1. Изоляции разработки новых функций
2. Параллельной работы нескольких разработчиков
3. Экспериментов без риска сломать основной код
4. Организации процесса ревью через Pull Request
5. Поддержки разных версий продукта
